Качество и запуск

Проверки проекта

Здесь только команды запуска, что именно они проверяют и где смотреть результат.

Unit и integration tests

.\.venv\Scripts\python.exe -m pytest

Проверяет: unit tests, integration tests, coverage API и collector.

Смотреть результат: в терминале `passed`, `failed`, `coverage %`.

Нагрузочный тест k6

$env:BASE_URL="https://research-platform-seven.vercel.app"
k6 run load_tests/k6_public_api.js

Проверяет: 100+ virtual users, RPS, latency, error rate.

Смотреть результат: в терминале `http_req_duration`, `http_req_failed`, `iterations`, `vus`.

Prometheus и Grafana

docker compose -f docker-compose.prod.yml up --build

Проверяет: monitoring stack и сбор метрик приложения.

Смотреть результат: `http://localhost:9090` для Prometheus и `http://localhost:3000` для Grafana.

GitHub Actions

GitHub -> Actions -> CI

Проверяет: линтер, тесты, coverage, Docker build.

Смотреть результат: статус workflow `passed` в GitHub Actions.
