Качество и запуск

Проверки проекта

Здесь собраны реальные команды и адреса для запуска тестов, нагрузки и мониторинга. Vercel показывает само приложение, а Prometheus и Grafana поднимаются отдельно локально через Docker.

Неделя 12. Тестирование

Что запускать:

.\.venv\Scripts\python.exe -m pytest

Что покрыто: API персонализации, collector, query params, metrics endpoint, авторизация через Supabase token.

Порог покрытия уже настроен в проекте: coverage >= 40%.

Неделя 13. Нагрузочное тестирование

Сценарий лежит в load_tests/k6_public_api.js.

$env:BASE_URL="https://research-platform-seven.vercel.app"
k6 run load_tests/k6_public_api.js

Что показать: RPS, latency, error rate и что сценарий поднимает нагрузку до 100+ virtual users.

Неделя 14. Мониторинг и деплой

Demo деплой: Vercel + Supabase PostgreSQL.

Метрики приложения: /api/v1/metrics

Prometheus и Grafana нужно поднимать локально, потому что Vercel не хостит такие сервисы внутри проекта.

docker compose -f docker-compose.prod.yml up --build

После запуска откройте:

App: `http://localhost:8000`

Prometheus: `http://localhost:9090`

Grafana: `http://localhost:3000`

Неделя 15. Финальная защита

Минимальный сценарий показа:

1. Открыть Vercel-приложение и показать поиск, профиль и сохранение данных.

2. Открыть Supabase и показать таблицы.

3. Запустить `pytest` и показать coverage.

4. Запустить `k6` и показать отчет нагрузки.

5. Поднять `docker compose -f docker-compose.prod.yml up --build`, открыть Prometheus и Grafana.

Что делать с Grafana

Grafana нужна не для Vercel-страницы, а для демонстрации мониторинга. Она читает метрики из Prometheus, а Prometheus забирает их из вашего FastAPI по адресу /api/v1/metrics.

Практический сценарий такой: поднимаете локально `app + prometheus + grafana` через `docker-compose.prod.yml`, в Grafana добавляете источник данных Prometheus с URL `http://prometheus:9090`, после чего строите 2-3 панели по метрикам запросов и времени ответа.

Для защиты этого достаточно: открыть Grafana, показать, что она подключена к Prometheus, затем открыть графики после нескольких запросов к приложению или после прогона `k6`.
