# Research Intelligence Platform

## О проекте

**Research Intelligence Platform** - это backend-first сервис поверх **OpenAlex API**, который выступает как инженерная и аналитическая надстройка над открытыми метаданными научных публикаций.

Система автоматически собирает данные о:

- научных работах;
- авторах;
- цитированиях;
- темах и научных областях;
- динамике публикационной активности.

После сбора данные сохраняются в **PostgreSQL**, проходят нормализацию и становятся доступны через собственный **domain-specific REST API** и веб-интерфейс на **FastAPI + Jinja2 + Tailwind CDN**.

Ключевая ценность проекта не в простом проксировании OpenAlex, а в создании собственного слоя аналитики:

- топ авторов по теме;
- тренды по годам;
- рост популярности темы;
- распределение публикаций;
- метрики цитируемости и impact;
- кеширование результатов;
- персонализированные функции пользователя.

Итог: это не просто клиент к внешнему API, а полноценный backend-сервис с хранением, обработкой, поиском, аналитикой, безопасностью, мониторингом и DevOps-процессами.

---

## Бизнес-идея

Платформа помогает исследователям, студентам, аналитикам и научным командам:

- быстро находить публикации по теме, автору и году;
- анализировать, какие направления науки растут быстрее;
- видеть топ авторов и наиболее цитируемые работы;
- отслеживать популярность тем по временным периодам;
- сохранять избранные статьи и собственные подборки;
- повторно использовать сохраненные поисковые запросы;
- работать быстрее за счет локального кеша, а не постоянной зависимости от OpenAlex в реальном времени.

## Архитектурная идея

Проект строится как набор логических слоев:

1. **Collector**
   Забирает данные из OpenAlex API по расписанию, с retry, rate-limit awareness и логированием ошибок.

2. **Storage**
   Хранит сырые и нормализованные данные в PostgreSQL, обеспечивает уникальность записей и миграции схемы.

3. **Processor**
   Очищает данные, выделяет ключевые слова, нормализует авторов, годы, темы и подготавливает агрегаты.

4. **Analytics Layer**
   Строит доменные метрики и выборки:
   топ авторов, тренды тем, распределения по годам, citations/impact.

5. **REST API**
   Выдает данные клиентам, поддерживает фильтрацию, поиск, пагинацию, экспорт и автодокументацию.

6. **Auth & Personalization**
   Управляет пользователями, JWT, избранным, подписками, коллекциями и сохраненными запросами.

7. **Delivery Layer**
   Покрывает контейнеризацию, локальный запуск, миграции и дальнейшую подготовку к CI/CD.

## Локальный запуск

### Предварительно

- Windows 11
- Docker Desktop
- Git

### Базовый сценарий для Windows PowerShell

```powershell
git clone <repo_url>
cd research-intelligence-platform
Copy-Item .env.example .env
docker compose up --build
```

Что делает запуск:

- собирает контейнер приложения;
- поднимает PostgreSQL;
- автоматически выполняет `alembic upgrade head`;
- запускает FastAPI с Jinja-интерфейсом.

После запуска должны быть доступны:

- `http://localhost:8000/` - главная страница
- `http://localhost:8000/ingestion` - запуск ingestion из браузера
- `http://localhost:8000/publications` - каталог публикаций
- `http://localhost:8000/authors` - каталог авторов
- `http://localhost:8000/topics` - каталог тем
- `http://localhost:8000/analytics` - аналитический dashboard
- `http://localhost:8000/profile` - auth и personalization UI
- `http://localhost:8000/docs` - Swagger/OpenAPI
- `http://localhost:8000/api/v1/health` - healthcheck

Если у вас остался старый `PostgreSQL` volume от предыдущих этапов и схема конфликтует с новыми миграциями, используйте:

```powershell
docker compose down -v
docker compose up --build
```

## Как проверять через браузер

### 1. Ingestion

Откройте `/ingestion` и запустите, например:

- `entity = works`
- `search = machine learning`
- `filter = publication_year:2024`
- `per_page = 25`
- `pages = 1`

После ответа обновите страницу и перейдите в каталоги.

### 2. Каталоги

- `/publications`
- `/authors`
- `/topics`

### 3. Аналитика

- `/analytics`
- `/topics/{id}`
- `/authors/{id}`
- `/publications/{id}`

### 4. Auth и personalization

Откройте `/profile`:

- зарегистрируйте пользователя;
- выполните логин;
- нажмите защищенные кнопки `me`, `favorites`, `saved-searches`, `collections`, `subscriptions`.

Для POST-проверок personalization удобно использовать еще и `/docs`.
