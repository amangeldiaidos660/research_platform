# Functional Specification

## Краткое описание функциональности

Проект представляет собой аналитическую платформу по научным публикациям на основе данных **OpenAlex**. Система собирает метаданные публикаций, авторов, цитирований и тем, сохраняет их в собственной БД, обрабатывает и предоставляет через REST API и веб-интерфейс.

Функционально платформа делится на 5 крупных блоков:

1. сбор данных;
2. поиск и просмотр данных;
3. аналитика;
4. пользовательские функции;

---

## Функциональные модули

### 1. Сбор данных

Назначение:
автоматически загружать данные из OpenAlex и поддерживать локальную базу в актуальном состоянии.

Что включает:

- импорт публикаций по теме, запросу или фильтрам;
- импорт авторов и связанных сущностей;
- регулярное обновление по расписанию;
- защита от дублей;
- журнал задач импорта.

### 2. Каталог публикаций

Назначение:
давать пользователю удобный поиск и просмотр научных работ.

Что включает:

- список публикаций;
- карточку публикации;
- фильтрацию по году, теме, автору, цитируемости;
- пагинацию;
- быстрый поиск по ключевым словам.

### 3. Каталог авторов

Назначение:
давать доступ к профилям авторов и их публикационной активности.

Что включает:

- список авторов;
- профиль автора;
- список публикаций автора;
- основные метрики автора;
- фильтрацию по темам и годам.

### 4. Аналитика

Назначение:
предоставлять ценность, которой нет в OpenAlex в виде готового инженерного слоя.

Что включает:

- топ авторов по теме;
- топ статей по цитируемости;
- тренды публикаций по годам;
- рост популярности темы;
- распределение публикаций по направлениям;
- агрегированные citation/impact метрики.

### 5. Пользовательский кабинет

Назначение:
добавить персонализацию и сделать платформу не просто каталогом, а рабочим инструментом.

Что включает:

- регистрация и вход;
- избранные статьи;
- сохраненные запросы;
- персональные подборки;
- подписки на темы;
- заготовка под рекомендации.

### 6. REST API

Назначение:
обеспечить программный доступ к данным платформы.

Что включает:

- публичные endpoints для чтения;
- приватные endpoints для пользовательских данных;
- JWT-аутентификацию;
- Swagger/OpenAPI документацию.

## Страницы и функциональность по страницам

### 1. Главная страница `/`

Что будет на странице:

- краткое описание платформы;
- форма глобального поиска;
- быстрые переходы в публикации, авторов и аналитику;
- блок с популярными темами;
- блок с топ статьями или трендами.

### 2. Страница публикаций `/publications`

Что будет на странице:

- список публикаций;
- фильтры по году, теме, автору, цитируемости;
- строка поиска;
- сортировка;
- пагинация.

### 3. Страница публикации `/publications/{id}`

Что будет на странице:

- полное название публикации;
- аннотация или краткое описание;
- год публикации;
- авторы;
- темы;
- citation metrics;
- ссылка на источник;
- кнопка добавления в избранное.

### 4. Страница авторов `/authors`

Что будет на странице:

- список авторов;
- поиск по имени;
- фильтры;
- краткие карточки с числом публикаций и цитирований.

### 5. Профиль автора `/authors/{id}`

Что будет на странице:

- имя автора;
- affiliations/institutions;
- список публикаций;
- темы автора;
- метрики автора;
- графики активности по годам.

### 6. Страница тем `/topics`

Что будет на странице:

- список тем и научных направлений;
- фильтрация;
- переход в аналитику по теме;
- краткие показатели популярности.

### 7. Страница аналитики `/analytics`

Что будет на странице:

- общие аналитические панели;
- топ авторов;
- топ статьи;
- распределение публикаций;
- динамика по годам;
- популярные темы.

### 8. Аналитика по теме `/analytics/topics/{id}`

Что будет на странице:

- описание темы;
- динамика публикаций по годам;
- рост или падение популярности;
- топ авторов по теме;
- самые значимые публикации по теме.

### 9. Аналитика по автору `/analytics/authors/{id}`

Что будет на странице:

- статистика публикаций автора;
- цитируемость;
- распределение работ по годам;
- основные темы;
- вклад автора в конкретные направления.

### 10. Избранное `/favorites`

Что будет на странице:

- список сохраненных пользователем публикаций;
- быстрый переход к карточкам;
- удаление из избранного;
- фильтрация по темам и дате добавления.

### 11. Сохраненные запросы `/saved-searches`

Что будет на странице:

- список сохраненных поисковых запросов;
- повторный запуск запроса;
- удаление и редактирование названия запроса.

### 12. Подборки `/collections`

Что будет на странице:

- список пользовательских подборок;
- создание новой подборки;
- добавление публикаций в подборку;
- просмотр содержимого подборки.

### 13. Подписки на темы `/subscriptions`

Что будет на странице:

- список подписок пользователя;
- управление интересующими темами;
- настройка будущих уведомлений или обновлений.

### 14. Вход `/login`

Что будет на странице:

- форма входа;
- получение JWT;
- переход в личный кабинет.

### 15. Регистрация `/register`

Что будет на странице:

- форма регистрации;
- создание учетной записи;
- первичная настройка интересов пользователя.

### 16. Профиль пользователя `/profile`

Что будет на странице:

- базовые данные пользователя;
- избранное;
- сохраненные запросы;
- подборки;
- подписки.

### 17. Системная страница статуса `/system`

Что будет на странице:

- health status;
- информация о последнем импорте;
- статус очередей и scheduler;
- ссылки на Swagger, Prometheus, Grafana.

## Основные API-группы

### Public API

- `GET /api/v1/publications`
- `GET /api/v1/publications/{id}`
- `GET /api/v1/authors`
- `GET /api/v1/authors/{id}`
- `GET /api/v1/topics`
- `GET /api/v1/analytics/trends`
- `GET /api/v1/analytics/top-authors`
- `GET /api/v1/analytics/top-publications`

### Auth API

- `POST /api/v1/auth/register`
- `POST /api/v1/auth/login`
- `POST /api/v1/auth/refresh`

### User API

- `GET /api/v1/me`
- `GET /api/v1/favorites`
- `POST /api/v1/favorites/{publication_id}`
- `DELETE /api/v1/favorites/{publication_id}`
- `GET /api/v1/saved-searches`
- `POST /api/v1/saved-searches`
- `GET /api/v1/collections`
- `POST /api/v1/collections`
- `GET /api/v1/subscriptions`
- `POST /api/v1/subscriptions`
